## نبذة عن البرنامج

<div dir="rtl">

برنامج **ذاتية الموظفين** هو نظام إدارة موارد بشرية متكامل مصمم خصيصاً للمكاتب الهندسية والجهات الحكومية. يتيح البرنامج:

- إدارة ملفات جميع الموظفين بشكل مركزي وآمن
- متابعة الحالة الوظيفية (فعّال، مؤرشف، إجازة، نقل، استقالة)
- رفع وتنظيم المستندات الخاصة بكل موظف (PDF، صور)
- استيراد البيانات من Excel وتصديرها بصيغ متعددة (Excel / Word)
- سجل تدقيق كامل لجميع العمليات
- لوحة تحكم تفاعلية بالإحصائيات والرسوم البيانية
- **بوت واتساب ذكي** يتيح للموظفين الاستعلام عن بياناتهم عبر واتساب مباشرةً

</div>

## المميزات الرئيسية

<div dir="rtl">

### 👤 إدارة الموظفين
- إضافة وتعديل وحذف بيانات الموظفين (حذف ناعم)
- بحث وفلترة متقدمة (بالاسم، الرقم الوطني، الاختصاص، الحالة)
- عرض البطاقات أو الجدول (Cards / Table View)
- عمليات جماعية (Bulk Actions): تغيير حالة متعدد أو تصدير مختار
- استيراد دفعة واحدة من Excel

### 📁 إدارة الملفات والمستندات
- رفع مستندات الموظفين (PDF، صور) — 10MB حداً أقصى للملف
- إنشاء بطاقة موظف تلقائياً بصيغة Word (docx)
- تصدير Excel مخصص مع اختيار الأعمدة والموظفين

### 📊 لوحة التحكم
- إحصائيات فورية: إجمالي الموظفين، فعّالون، مؤرشفون، لديهم ملفات
- رسم بياني للاختصاصات الأكثر شيوعاً (أعلى 7)
- مؤشرات أداء ديناميكية مع تأثير عداد متحرك
- Skeleton Loaders أثناء التحميل

### 🤖 بوت واتساب الذكي
- استعلام الموظفين عن بياناتهم وملفاتهم عبر واتساب
- تفعيل وإيقاف بكودات خاصة لكل موظف
- ذاكرة محادثة مستقلة لكل مستخدم
- إيقاف تلقائي بعد 10 دقائق من الخمول
- حماية من سرقة الجلسة (Session Hijacking Protection)
- لا يرد مطلقاً من خارج قاعدة البيانات

### 🔒 الأمان والإدارة
- نظام مصادقة بالجلسات مع حماية ضد هجمات Brute Force (5 محاولات / 15 دقيقة)
- تحذير انتهاء الجلسة قبل دقيقة مع خيار التمديد
- سجل تدقيق كامل لكل عملية في النظام (CREATE، UPDATE، DELETE، LOGIN، LOGOUT)
- إدارة المستخدمين بالأدوار (مدير / موظف)
- نظام مفاتيح API لصلاحيات البرمجية (human / machine)

### 🌙 تجربة المستخدم
- واجهة عربية بالكامل مع دعم RTL
- وضع داكن / فاتح (Dark / Light Mode)
- تصميم متجاوب مع الشاشات المختلفة
- النسخ الاحتياطي واستعادة البيانات (JSON)

</div>

## نظام بوت واتساب

<div dir="rtl">

### كيف يعمل البوت

```
الموظف يرسل رسالة واتساب
        ↓
n8n Webhook يستقبل الرسالة
        ↓
POST /api/v1/bot/check-auth
  ← action: "activated"     → رسالة ترحيب
  ← action: "deactivated"   → رسالة وداع
  ← action: "auto_deactivated" → إشعار انتهاء الجلسة
  ← action: "message"       → الذكاء الاصطناعي
  ← action: "unauthorized"  → صامت (لا رد)
```

### إدارة الجلسات

| الحالة | الوصف |
|--------|-------|
| غير مفعّل | المستخدم يرسل كود التفعيل → تبدأ الجلسة |
| مفعّل | كل رسالة تُحدِّث `lastInteraction` |
| خمول 10 دقائق | إيقاف تلقائي + إشعار |
| كود الإيقاف | إيقاف يدوي فوري |

### حماية سرقة الجلسة (LID Hijacking)

إذا كان الموظف A لديه WhatsApp LID مسجّل، وحاول شخص B استخدام كود الموظف A من جهاز مختلف → **يُرفض الطلب تلقائياً** ويبقى الموظف A يتلقى الردود بشكل طبيعي.

### إعادة تعيين الجهاز (لتغيير الهاتف)

1. في صفحة **المستخدمون** → تبويب **مستخدمو البوت**
2. اضغط أيقونة الهاتف البنفسجية بجانب الموظف
3. أكّد الحذف → الموظف يستطيع الآن التفعيل من جهازه الجديد

## إعداد ورك فلو n8n

<div dir="rtl">

### ملف الورك فلو
الملف الجاهز للاستيراد: `docs/workflows/Sidawi_AI_Health_V22.json`

### متطلبات n8n
- خادم n8n مشغَّل
- بيانات اعتماد **Google Gemini API** مُعدَّة في n8n
- خادم واتساب على `http://172.17.0.1:8082` (أو تحديث الروابط)

### مكونات الورك فلو

| Node | الوظيفة |
|------|---------|
| `Sidawi_Health_WH` | Webhook يستقبل الرسائل من واتساب |
| `Verify Identity1` | يستدعي `/api/v1/bot/check-auth` للتحقق |
| `Route by Action1` | يوجّه حسب النتيجة (4 مسارات) |
| `WhatsApp Welcome1` | رسالة ترحيب عند التفعيل |
| `WhatsApp Goodbye1` | رسالة وداع عند الإيقاف |
| `WhatsApp Auto Timeout1` | إشعار إيقاف تلقائي |
| `AI Agent Router1` | وكيل الذكاء الاصطناعي (Gemini) |
| `fetch_employee_database1` | أداة جلب قاعدة البيانات |
| `generate_word_link1` | أداة توليد ملف Word |
| `export_excel_tool` | أداة تصدير Excel |
| `Conversation Memory1` | ذاكرة المحادثة (مستقلة لكل مستخدم) |

### السيستم برومبت — القواعد الصارمة
1. **الأداة إلزامية**: استدعاء `fetch_employee_database` قبل أي رد
2. **الموظف غير الموجود**: الرد الحرفي الوحيد: *"لا يوجد موظف بهذا الاسم أو الرقم في قاعدة بيانات المديرية."*
3. **المستندات**: من `direct_url` في البيانات فقط — لا اختراع للروابط
4. **ملف Word**: يستلزم `nationalId` من الأداة أولاً
5. **خارج النطاق**: *"أنا متخصص في بيانات موظفي المديرية فقط."*

## نظام مفاتيح API

<div dir="rtl">

### أنواع المفاتيح

| النوع | الاستخدام | تسجيل الدخول البشري |
|-------|----------|---------------------|
| `human` | للمستخدمين البشريين | مسموح |
| `machine` | للبرمجيات (n8n) | محظور (403) |

### إنشاء مفتاح لـ n8n
1. سجّل دخولك كمدير
2. اذهب إلى **الإعدادات** → **مفاتيح API**
3. اضغط **إنشاء مفتاح جديد**
4. اختر النوع: `machine` والوصف: `n8n`
5. انسخ المفتاح **مرة واحدة فقط** (لن يُعرض مجدداً)

### وضع Bootstrap
عند أول تشغيل وجدول مفاتيح API فارغ، يُسمح بتسجيل الدخول دون مفتاح لإنشاء المفتاح الأول.

## الأمان والحماية

<div dir="rtl">

| الميزة | التفاصيل |
|--------|----------|
| حماية Brute Force | 5 محاولات دخول / 15 دقيقة |
| تشفير كلمات المرور | scrypt (أقوى من bcrypt) |
| الجلسات | مخزّنة في PostgreSQL عبر `connect-pg-simple` |
| Cookie | `httpOnly: true`، `secure: true` في الإنتاج |
| CSRF | الجلسات + Same-Site Cookies |
| API Keys | `crypto.randomBytes(32)` — 64 رمز hex |
| Session Hijacking (Bot) | رفض استبدال LID موظف آخر |
| سجل التدقيق | تسجيل جميع العمليات مع المستخدم والوقت |

</div>
