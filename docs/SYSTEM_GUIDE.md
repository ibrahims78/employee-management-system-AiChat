## 1. نبذة عامة عن البرنامج

برنامج **ذاتية الموظفين** نظام متكامل لإدارة الموارد البشرية مصمّم خصيصاً للقسم الهندسي الصحي. يجمع بين:

- **لوحة تحكم ويب** بواجهة عربية كاملة
- **بوت ذكي متعدد القنوات** (واتساب + تيليغرام) مدعوم بالذكاء الاصطناعي
- **نظام مستندات** لرفع وتنظيم ملفات الموظفين
- **تصدير واستيراد** بصيغ Excel وWord
- **سجل تدقيق** كامل لكل عملية في النظام

## 4. الصفحات والوظائف

### 4.1 صفحة تسجيل الدخول (`/login`)

**الوظائف:**
- نموذج دخول بالاسم وكلمة المرور
- حماية Brute Force: 5 محاولات خاطئة = حظر 15 دقيقة
- في حالة `apiKeyRequired: true`، يُطلب إضافة `x-api-key` في الـ header
- في وضع Bootstrap (جدول `api_keys` فارغ): الدخول بدون مفتاح

---

### 4.2 لوحة التحكم (`/`)

**الوظائف:**
- **بطاقات الإحصائيات الآنية:**
  - إجمالي الموظفين (الفعّالون)
  - الموظفون على رأس العمل
  - موظفون بإجازة بلا أجر / في انتقال
  - الموظفون الذين لديهم مستندات مرفوعة
- **رسم بياني للاختصاصات:** أعلى 7 تخصصات (Recharts BarChart)
- **عداد متحرك:** الأرقام تتحرك من 0 إلى القيمة الفعلية عند التحميل
- **Skeleton Loaders:** عرض هياكل رمادية أثناء تحميل البيانات

---

### 4.3 إدارة الموظفين (`/employees`)

**طريقتا العرض:**
- **Cards View:** بطاقات جميلة لكل موظف مع صورة مصغّرة ومعلومات سريعة
- **Table View:** جدول مضغوط بكل بيانات الموظفين مع ترتيب وفلترة

**الفلاتر المتاحة:**
- البحث النصي (الاسم، الرقم الوطني، الاختصاص)
- الحالة الوظيفية (مثبت / عقد)
- الوضع الحالي (على رأس عمله / إجازة / ...)
- الاختصاص (من قائمة ديناميكية)
- يشمل المؤرشَفين (toggle)

**العمليات على الموظف المفرد:**
- إضافة موظف جديد (نموذج شامل)
- عرض تفاصيل الموظف (صفحة مخصصة)
- تعديل بيانات الموظف
- أرشفة / إلغاء أرشفة (حذف ناعم)
- رفع مستندات (PDF، صور — حد 10MB)
- حذف مستندات
- تنزيل بطاقة Word
- تصدير Excel

**العمليات الجماعية (Bulk Actions):**
- تحديد مجموعة موظفين بصناديق الاختيار
- تغيير الحالة لمجموعة دفعة واحدة
- تصدير Excel للمحددين فقط

**استيراد من Excel:**
- رفع ملف Excel بعمود واحد أو أكثر
- التحقق من البيانات قبل الاستيراد
- تقرير بنتائج الاستيراد (ناجح / مرفوض / مكرر)

### 4.4 تفاصيل الموظف (`/employees/:id`)

**المعلومات المعروضة:**
- جميع البيانات الشخصية والوظيفية
- تواريخ التوظيف والمباشرة
- قائمة المستندات مع إمكانية الرفع والحذف والتنزيل

### 4.5 إدارة المستخدمين (`/users`)

**تبويبان:**

**تبويب 1: مستخدمو النظام**
- قائمة المستخدمين البشريين (مدير / موظف)
- إضافة مستخدم جديد
- تعديل: تغيير الاسم، كلمة المرور، الدور
- حذف المستخدم (مع تحذير: يحذف سجلاته في التدقيق)

**تبويب 2: مستخدمو البوت**
- قائمة الموظفين المسجّلين في بوت واتساب
- إضافة مستخدم بوت (الاسم، رقم الهاتف، أكواد التفعيل/الإيقاف)
- مؤشر حالة البوت (نشط / خامل)
- زر إعادة تعيين الجهاز (مسح `whatsappLid`)
- حذف مستخدم البوت

**الوظائف:**

### 4.7 الإعدادات (`/settings`)

**تبويبان:**

**تبويب 1: إعدادات النظام**
- اسم الجهة / المديرية
- إعدادات أخرى قابلة للتخصيص
- نسخة احتياطية (تصدير JSON لجميع بيانات النظام)
- استعادة من نسخة احتياطية (استيراد JSON)

**تبويب 2: مفاتيح API**
- قائمة المفاتيح الموجودة (القيمة مُخفاة)
- إنشاء مفتاح جديد (human / machine)
- تعطيل / تفعيل مفتاح
- حذف مفتاح
- عند الإنشاء: القيمة تُعرض مرة واحدة فقط

## 5. نظام المصادقة والأمان

### 5.1 تسجيل الدخول

```
المستخدم يُدخل username + password
          ↓
Passport.js LocalStrategy
          ↓
البحث في جدول users بـ username
          ↓
مقارنة كلمة المرور بـ scrypt
          ↓
نجاح → إنشاء جلسة في PostgreSQL
فشل → زيادة عداد المحاولات الفاشلة
```

### 5.2 انتهاء الجلسة

- **مدة الجلسة:** 5 دقائق من آخر طلب
- **التجديد التلقائي:** كل طلب يُجدّد العداد (`rolling: true`)
- **تحذير قبل الانتهاء:** قبل دقيقة من انتهاء الجلسة تظهر نافذة تحذير
- **تمديد الجلسة:** المستخدم يستطيع الضغط على "تمديد" لإعادة تعيين العداد

### 5.3 حماية الـ Brute Force

```javascript
rateLimit({
  windowMs: 15 * 60 * 1000,  // 15 دقيقة
  max: 5,                      // 5 محاولات فقط
  message: "حظر 15 دقيقة"
})
```

### 5.4 Cookies الآمنة

| الخاصية | القيمة في الإنتاج |
|---------|------------------|
| `httpOnly` | `true` (JavaScript لا يستطيع القراءة) |
| `secure` | `true` (HTTPS فقط) |
| `sameSite` | `lax` (حماية CSRF) |

### 5.5 مصادقة API

كل طلب لمسارات البوت يحتاج هيدر:
```
x-api-key: <مفتاح machine>
```

أو عبر query string:
```
?_t=<مفتاح>
```

## 6. نظام الملفات والمستندات

### 6.1 رفع الملفات

| الخاصية | القيمة |
|---------|--------|
| مكتبة الرفع | Multer (disk storage) |
| مجلد التخزين | `storage/uploads/` |
| الأنواع المسموح بها | PDF، JPEG، PNG، DOCX، XLSX |
| الحجم الأقصى | 10MB لكل ملف |
| تسمية الملف | `{timestamp}-{random}.{ext}` |

### 6.2 خدمة الملفات بحماية API Key

```
GET /api/v1/files/uploads/1710000000-abc.pdf?_t=<api-key>
```

- الوصول محمي بمفتاح API
- يعمل في المتصفح المباشر وفي واتساب عبر الرابط

### 6.3 توليد Word

```
GET /api/v1/bot/generate-word-link?nationalId=010XXXXXXXX
```

- يولّد ملف `.docx` بتصميم جدول
- يُحفظ في `storage/uploads/word_exports/`
- اسم الملف: `بطاقة_{nationalId}_{YYYYMMDD}_{HHmmss}.docx`
- الرابط المُعاد يحتوي على مفتاح API للوصول المباشر

### 6.4 تصدير Excel المخصص

```
GET /api/v1/bot/generate-custom-excel
```

**معاملات الفلترة (كلها اختيارية):**

| المعامل | الوصف |
|---------|-------|
| `status` | وضع العامل (على رأس عمله / إجازة / نقل...) |
| `category` | الفئة (طبي / تمريض / إداري / تقني...) |
| `gender` | الجنس (ذكر / أنثى) |
| `employmentStatus` | نوع التوظيف (دائم / مؤقت / عقد) |
| `assignedWork` | العمل المكلف به |
| `search` | بحث نصي بالاسم أو الرقم الوطني أو الجوال |
| `columns` | أعمدة مختارة مفصولة بفاصلة |
| `title` | عنوان مخصص للملف |

- إذا لم يُحدَّد أي معامل: يُصدَّر جميع الموظفين بجميع الأعمدة
- يُحفظ في `storage/uploads/excel_exports/`
- اسم الملف: `تقرير_مخصص_{YYYYMMDD}_{HHmmss}.xlsx`

> **ملاحظة:** النقطة القديمة `GET /api/v1/bot/generate-excel-link` لا تزال موجودة للتوافق العكسي وتُصدِّر دائماً جميع الموظفين بجميع الأعمدة.

## 12. ملاحظات المطوّر

### نقاط مهمة في الكود

1. **`server/auth.ts`**: يحتوي على بذر مفتاح n8n تلقائياً عند الإقلاع
2. **`server/routes.ts`**: جميع مسارات API — نقطة المرجع الرئيسية
3. **`script/build.ts`**: يحتوي على نسخ `table.sql` للإنتاج (إصلاح مهم)
4. **`shared/schema.ts`**: أنواع Drizzle + Zod — المصدر الوحيد للحقيقة
5. **URL الديناميكي**: `req.protocol + req.get("host")` بدلاً من متغير ثابت

### قرارات تصميمية

- **Soft Delete**: الموظفون لا يُحذفون فعلياً → يمكن استعادتهم
- **No Migration Files**: `drizzle-kit push` مباشرة → أبسط للتطوير المتسارع
- **JSONB for documents**: مصفوفة المسارات في عمود واحد → أبسط من جدول منفصل
- **Machine API Key**: مفتاح ثابت تلقائي لـ n8n → لا يحتاج إعداداً يدوياً
- **Rolling Sessions**: تجديد الجلسة مع كل طلب → لا يصطدم المستخدم بانتهاء مفاجئ
