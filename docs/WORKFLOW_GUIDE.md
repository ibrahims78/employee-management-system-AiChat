## الغاية من الورك فلو

يهدف هذا الورك فلو إلى ربط **واتساب** بـ **نظام إدارة الموظفين** عبر الذكاء الاصطناعي (Google Gemini)، مما يُتيح لكل موظف مسجّل في النظام الاستعلام عن بياناته الوظيفية ومستنداته مباشرةً من هاتفه عبر واتساب، دون الحاجة للدخول إلى أي موقع أو تطبيق.

### ما يستطيع البوت تقديمه للمستخدم

| نوع الاستعلام | مثال |
|---------------|------|
| البيانات الشخصية | الاسم الكامل، اسم الأب، الأم، مكان وتاريخ الميلاد |
| البيانات الوظيفية | المسمى الوظيفي، الفئة، الوضع الوظيفي (مثبت/عقد) |
| الحالة الحالية | على رأس عمله / إجازة بلا أجر / نقل / استقالة |
| تواريخ التوظيف | تاريخ قرار التعيين، أول مباشرة بالدولة/المديرية/القسم |
| الشهادة والاختصاص | نوع الشهادة، مجال التخصص |
| بيانات التواصل | رقم الجوال، العنوان |
| رقم شام كاش | رقم الدفع المالي |
| المستندات | روابط تنزيل مباشرة للوثائق المرفوعة (PDF، صور) |
| بطاقة Word | توليد بطاقة الموظف الرسمية كملف Word قابل للتنزيل |
| تقرير Excel | تصدير قاعدة البيانات الكاملة لجميع الموظفين كملف Excel |
| الإحصائيات | العدد الإجمالي، التوزيع حسب الاختصاص/الحالة/الفئة |
| الاستعلام العام | أي معلومة عن أي موظف في قاعدة البيانات |

### ما لا يستطيع البوت فعله

- **لا يخترع معلومات** — إذا لم يجد الموظف يقول ذلك صراحةً
- **لا يرد على مواضيع خارج النطاق** — غير متخصص بغير بيانات المديرية
- **لا ينشئ روابط وهمية** — المستندات من قاعدة البيانات فقط

**المُدخلات التي يُرسلها:**
```json
{
  "phoneNumber": "{{ $json.body.from }}",
  "activationCode": "{{ $json.body.content }}"
```

**الردود المحتملة من الخادم:**

| القيمة في `action` | المعنى |
|--------------------|---------|
| `activated` | أرسل المستخدم كود التفعيل الصحيح → البوت مُفعَّل الآن |

| `message` | البوت مُفعَّل والرسالة محتوى عادي → أرسل للذكاء الاصطناعي |
| `unauthorized` | المستخدم غير مسجّل أو كود خاطئ → تجاهل صامت |

**بيانات إضافية في الرد:**
```json
{
  "authorized": true,
  "action": "message",
  "full_name": "اسم الموظف الكامل",
  "is_bot_active": true
}
```

## إعداد الورك فلو — متطلبات النشر

### 1. بيانات الاعتماد المطلوبة في n8n

| الخدمة | نوع الاعتماد | ملاحظة |
|---------|-------------|--------|
| Google Gemini | `googlePalmApi` | مفتاح Google AI Studio |

### 2. إعداد خادم واتساب

روابط الإرسال تشير إلى `http://172.17.0.1:8082/send-text`. هذا يعني خادم واتساب (مثل WPPConnect أو Baileys) يعمل على:
- الآي بي: `172.17.0.1` (Docker Gateway — الشبكة الداخلية)
- المنفذ: `8082`

إذا كان خادم واتساب على عنوان مختلف، يجب تحديث الروابط في العقد: `WhatsApp Welcome1` و `WhatsApp Goodbye1` و `WhatsApp Auto Timeout1` و `WhatsApp AI Output1`.

### 3. استيراد الورك فلو

1. افتح n8n
2. اضغط **+** لإنشاء ورك فلو جديد
3. من القائمة اختر **Import from file**
4. ارفع ملف: `docs/workflows/Sidawi_AI_Health_V22.json`
5. احفظ وفعّل الورك فلو

## نظام الجلسات والأمان

### دورة حياة جلسة البوت

```
الموظف مسجّل في قاعدة بيانات bot_users
         ↓
يُرسل كود التفعيل الخاص به
         ↓
النظام يتحقق: هل الكود صحيح؟ هل الجهاز مسجَّل؟
         ↓
إذا صحيح → isBotActive = true، تسجيل LID الجهاز
         ↓
الرسائل التالية تُحوَّل للذكاء الاصطناعي
         ↓
بعد 10 دقائق خمول → isBotActive = false (تلقائياً)
  أو
يُرسل كود الإيقاف → isBotActive = false (يدوياً)
```

### حماية سرقة الجلسة (LID Hijacking Protection)

إذا حاول شخص يملك كود موظف آخر تفعيل البوت من جهازه:
- **الشرط:** الموظف لديه LID مسجّل (sجهاز آخر) + الطلب الجديد يحمل LID مختلف
- **النتيجة:** `action: unauthorized` — رفض صامت
