## الغاية من الورك فلو

يهدف هذا الورك فلو إلى ربط **واتساب** بـ **نظام إدارة الموظفين** عبر الذكاء الاصطناعي (Google Gemini)، مما يُتيح لكل موظف مسجّل في النظام الاستعلام عن بياناته الوظيفية ومستنداته مباشرةً من هاتفه عبر واتساب، دون الحاجة للدخول إلى أي موقع أو تطبيق.

### ما يستطيع البوت تقديمه للمستخدم

| نوع الاستعلام | مثال |
|---------------|------|
| البيانات الشخصية | الاسم الكامل، اسم الأب، الأم، مكان وتاريخ الميلاد |
| البيانات الوظيفية | المسمى الوظيفي، الفئة، الوضع الوظيفي (مثبت/عقد) |
| الحالة الحالية | على رأس عمله / إجازة بلا أجر / نقل / استقالة |
| تواريخ التوظيف | تاريخ قرار التعيين، أول مباشرة بالدولة/المديرية/القسم |
| الشهادة والاختصاص | نوع الشهادة، مجال التخصص |
| بيانات التواصل | رقم الجوال، العنوان |
| رقم شام كاش | رقم الدفع المالي |
| المستندات | روابط تنزيل مباشرة للوثائق المرفوعة (PDF، صور) |
| بطاقة Word | توليد بطاقة الموظف الرسمية كملف Word قابل للتنزيل |
| تقرير Excel مخصص | تصدير ملف Excel مخصص بالفلاتر والأعمدة التي يختارها الموظف |
| الإحصائيات | العدد الإجمالي، التوزيع حسب الاختصاص/الحالة/الفئة |
| الاستعلام العام | أي معلومة عن أي موظف في قاعدة البيانات |

### ما لا يستطيع البوت فعله

- **لا يخترع معلومات** — إذا لم يجد الموظف يقول ذلك صراحةً
- **لا يرد على مواضيع خارج النطاق** — غير متخصص بغير بيانات المديرية
- **لا ينشئ روابط وهمية** — المستندات من قاعدة البيانات فقط

```
[واتساب] → رسالة من موظف

[Sidawi_Health_WH] → استقبال الرسالة عبر Webhook
     ↓
[Verify_Identity] → التحقق من الهوية والجلسة
     ↓
[Route_by_Action] → توجيه حسب نوع الحدث
     ↓
 ┌───────────────────────────────────────────────────┐
 │            أربعة مسارات                          │
 ├─────────────────┬──────────────┬────────────────────┤
 │ تفعيل          │ إيقاف يدوي  │ إيقاف تلقائي      │ رسالة ذكاء
 │                │              │                    │
 ▼                ▼              ▼                    ▼
[WA_Welcome]  [WA_Goodbye] [WA_AutoTimeout]      [AI_Agent]
                                              ↓
                                    [fetch_employee_database]
                                    [generate_word_link]
                                    [export_excel_tool]
                                    [Conversation_Memory]
                                    [Gemini_Model]
                                              ↓
                                    [WA_AI_Output]
```

### 2. Verify_Identity — التحقق من الهوية

| الخاصية | القيمة |
|---------|--------|
| النوع | `n8n-nodes-base.httpRequest` |
| الإصدار | 4.1 |
| طريقة HTTP | `POST` |
| الرابط | `https://employee-management-system-ai-chat--alsid2225.replit.app/api/v1/bot/check-auth` |
| المصادقة | `x-api-key: 3477e2bd6616a95eb2dcbb3a9e39b663fddab5a90fe7d71cdd45a7b34040fca4` |

**المُدخلات التي يُرسلها:**
```json
{
  "phoneNumber": "{{ $json.body.from }}",
  "activationCode": "{{ $json.body.content }}"
}
```

**الردود المحتملة من الخادم:**

| القيمة في `action` | المعنى |
|--------------------|---------|
| `activated` | أرسل المستخدم كود التفعيل الصحيح → البوت مُفعَّل الآن |
| `deactivated` | أرسل المستخدم كود الإيقاف → البوت متوقف |
| `auto_deactivated` | مرّت 5 دقائق خمول → أوقف النظام الجلسة تلقائياً |
| `message` | البوت مُفعَّل والرسالة محتوى عادي → أرسل للذكاء الاصطناعي |
| `unauthorized` | المستخدم غير مسجّل أو كود خاطئ → تجاهل صامت |

**بيانات إضافية في الرد:**
```json
{
  "authorized": true,
  "action": "message",
  "full_name": "اسم الموظف الكامل",
  "is_bot_active": true
}
```

### 10. export_excel_tool — أداة تصدير Excel المخصص

| الخاصية | القيمة |
|---------|--------|
| النوع | `@n8n/n8n-nodes-langchain.toolHttpRequest` |
| الإصدار | 1.1 |
| طريقة HTTP | `GET` |
| الرابط | `https://employee-management-system-ai-chat--alsid2225.replit.app/api/v1/bot/generate-custom-excel` |
| المُعاملات | جميعها اختيارية — يحددها الذكاء الاصطناعي بناءً على طلب المستخدم |

**المُعاملات المدعومة:**

| المُعامل | الوصف | مثال |
|---------|-------|-------|
| `status` | وضع العامل الحالي | على رأس عمله / إجازة / نقل |
| `category` | الفئة الوظيفية | طبي / تمريض / إداري / تقني |
| `gender` | الجنس | ذكر / أنثى |
| `employmentStatus` | نوع التوظيف | دائم / مؤقت / عقد |
| `assignedWork` | العمل المكلف به | اسم العمل |
| `search` | بحث نصي | اسم أو رقم وطني أو جوال |
| `columns` | أعمدة مختارة مفصولة بفاصلة | الاسم والكنية,رقم الجوال |
| `title` | عنوان مخصص للملف | تقرير الأطباء |

**الوصف للوكيل:**
> أداة تصدير بيانات الموظفين كملف Excel مخصص. تدعم الفلترة حسب الوضع والفئة والجنس ونوع التوظيف والعمل والبحث النصي، وتدعم اختيار أعمدة محددة فقط. إذا لم يُحدَّد أي معامل يُصدَّر جميع الموظفين بجميع الأعمدة.

**أمثلة على الاستخدام:**
- "صدِّر بيانات الموظفين إكسل" ← يصدِّر الكل بجميع الأعمدة
- "أريد إكسل موظفي التمريض فقط" ← يُفعِّل `category=تمريض`
- "أعطني إكسل الإناث على رأس عملهن" ← `gender=أنثى` + `status=على رأس عمله`
- "إكسل بالاسم والجوال والصفة فقط" ← يُمرِّر `columns` المحددة
- "جدول الأطباء الدائمين" ← `category=طبي` + `employmentStatus=دائم`

**الرد:**
```json
{
  "status": "success",
  "employeeCount": 45,
  "columnsCount": 3,
  "filtersApplied": "الفئة: تمريض | الجنس: أنثى",
  "downloadUrl": "https://...replit.app/api/v1/files/uploads/excel_exports/تقرير_مخصص_...xlsx?_t=...",
  "fileName": "تقرير_مخصص_20260318_143034.xlsx",
  "message": "تم إنشاء ملف Excel مخصص يحتوي على 45 موظف (3 عمود)..."
}
```

**الأعمدة المتاحة للاختيار:**
الاسم والكنية | اسم الأب | اسم الأم | مكان الولادة | تاريخ الولادة | محل ورقم القيد | الرقم الوطني | رقم شام كاش | الجنس | الشهادة | نوع الشهادة | الاختصاص | الصفة الوظيفية | الفئة | الوضع الوظيفي | رقم قرار التعيين | تاريخ قرار التعيين | أول مباشرة بالدولة | أول مباشرة بالمديرية | أول مباشرة بالقسم | وضع العامل الحالي | العمل المكلف به | رقم الجوال | العنوان | ملاحظات

## نظام الجلسات والأمان

### دورة حياة جلسة البوت

```
الموظف مسجّل في قاعدة بيانات bot_users
         ↓
يُرسل كود التفعيل الخاص به
         ↓
النظام يتحقق: هل الكود صحيح؟ هل الجهاز مسجَّل؟
         ↓
إذا صحيح → isBotActive = true، تسجيل LID الجهاز
         ↓
الرسائل التالية تُحوَّل للذكاء الاصطناعي
         ↓
بعد 5 دقائق خمول → isBotActive = false (تلقائياً)
  أو
يُرسل كود الإيقاف → isBotActive = false (يدوياً)
```

### حماية سرقة الجلسة (LID Hijacking Protection)

إذا حاول شخص يملك كود موظف آخر تفعيل البوت من جهازه:
- **الشرط:** الموظف لديه LID مسجّل (sجهاز آخر) + الطلب الجديد يحمل LID مختلف
- **النتيجة:** `action: unauthorized` — رفض صامت
- **الموظف الأصلي:** يستمر في استخدام البوت بشكل طبيعي

### إعادة تعيين الجهاز

عبر واجهة المستخدم (صفحة المستخدمون → مستخدمو البوت):
1. اضغط أيقونة الهاتف البنفسجية
2. أكّد الحذف → يُمسح `whatsappLid`
3. الموظف يستطيع الآن التفعيل من جهازه الجديد
