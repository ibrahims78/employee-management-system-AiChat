# دليل مفاتيح API وأكواد التفعيل

<div dir="rtl">

## أولاً: مفاتيح API للنظام

مفاتيح API هي رموز سرية طويلة (64 حرف hex) تُستخدم للوصول البرمجي إلى الخادم. تُدار من صفحة **الإعدادات → مفاتيح API**.

---

### المفاتيح المخزّنة حالياً في النظام

#### مفتاح n8n (machine)

| الحقل | القيمة |
|-------|--------|
| النوع | `machine` |
| الوصف | `n8n` |
| الحالة | نشط ✅ |
| المفتاح الكامل | `3477e2bd6616a95eb2dcbb3a9e39b663fddab5a90fe7d71cdd45a7b34040fca4` |

> **ملاحظة:** لا يوجد في النظام حالياً مفتاح من نوع `human`. يمكن إنشاء واحد من صفحة الإعدادات إذا احتجت وصولاً برمجياً بصلاحيات بشرية.

---

### نوعا مفاتيح API

| النوع | الرمز | من يستخدمه | تسجيل دخول بشري |
|-------|-------|-----------|-----------------|
| بشري | `human` | المطوّر أو تطبيقات خارجية | ✅ مسموح |
| آلة | `machine` | n8n أو بوت أو سكريبت | ❌ محظور (يُعيد 403) |

---

### كيف يعمل مفتاح API في النظام

#### طريقة الإرسال — خيارات متعددة

**1. عبر HTTP Header (الطريقة الموصى بها):**
```http
x-api-key: 3477e2bd6616a95eb2dcbb3a9e39b663fddab5a90fe7d71cdd45a7b34040fca4
```

**2. عبر Query String (للروابط المباشرة مثل تنزيل الملفات):**
```
https://...replit.app/api/v1/bot/master-query?_t=3477e2bd6616a95eb2dcbb3a9e39b663fddab5a90fe7d71cdd45a7b34040fca4
```

#### الكود في الخادم — `server/apiKeyAuth.ts`

```typescript
// يتحقق من المفتاح في كلا المصدرين
const apiKey = req.headers["x-api-key"] || req.query["_t"];

// البحث في قاعدة البيانات
const keyRecord = await storage.getApiKeyByValue(apiKey);

// التحقق من الشروط
if (!keyRecord || !keyRecord.isActive) → 401 Unauthorized
if (keyRecord.expiryDate && keyRecord.expiryDate < now) → 401 Expired
if (keyRecord.keyType === "machine" && req.path === "/api/auth/login") → 403 Forbidden
```

#### أين يُستخدم المفتاح في التطبيق

| المسار | يتطلب مفتاح API | نوع المفتاح |
|--------|----------------|------------|
| `POST /api/v1/bot/check-auth` | ✅ | machine |
| `POST /api/v1/bot/get-all-data` | ✅ | machine |
| `GET /api/v1/bot/master-query` | ✅ | machine |
| `GET /api/v1/bot/generate-word-link` | ✅ | machine |
| `GET /api/v1/bot/generate-excel-link` | ✅ | machine |
| `GET /api/v1/bot/generate-custom-excel` | ✅ | machine |
| `GET /api/v1/files/*` | ✅ | machine |
| `/api/auth/login` | human فقط | machine محظور |
| جميع مسارات `/api/*` الأخرى | جلسة متصفح | — |

---

### إنشاء مفتاح API جديد

1. سجّل دخولك بحساب **admin**
2. اذهب إلى **الإعدادات** → تبويب **مفاتيح API**
3. اضغط **إنشاء مفتاح جديد**
4. اختر النوع (**machine** لـ n8n، **human** للوصول البشري)
5. أضف وصفاً (مثل: `n8n` أو `تطبيق خارجي`)
6. اضغط **إنشاء** — **المفتاح يُعرض مرة واحدة فقط، احفظه فوراً**

---

### الأمان — تفاصيل تقنية

| الميزة | التفاصيل |
|--------|----------|
| طول المفتاح | 64 حرف hex = 256 bit |
| طريقة التوليد | `crypto.randomBytes(32).toString("hex")` |
| التخزين | نص عادي في قاعدة البيانات (لا تشفير) |
| العرض في الواجهة | مُخفّى — يظهر فقط لحظة الإنشاء |
| انتهاء الصلاحية | اختياري (تاريخ قابل للتحديد) |

## ثانياً: أكواد التفعيل والإلغاء للبوت

هذه ليست مفاتيح API بل **رموز نصية قصيرة** يُرسلها الموظف عبر واتساب لتفعيل أو إيقاف البوت.

---

### ما هي أكواد التفعيل والإلغاء؟

| نوع الكود | الاستخدام |
|-----------|----------|
| **كود التفعيل** | يُرسله الموظف عبر واتساب ليبدأ جلسة البوت |
| **كود الإلغاء** | يُرسله الموظف عبر واتساب ليوقف جلسة البوت |

---

### أين تُحدَّد الأكواد؟

في صفحة **المستخدمون** → تبويب **مستخدمو البوت** → **إضافة مستخدم بوت**:

```
الاسم الكامل:   محمد أحمد الخطيب
رقم الهاتف:     0991234567
كود التفعيل:    health_on_1234      ← يُرسله الموظف ليبدأ
كود الإلغاء:    health_off_1234     ← يُرسله الموظف ليوقف
```

> يمكن أن يكون الكود أي نص تريده: أرقام، حروف، أو مزيج. **يجب أن يكون فريداً** (لا يتشارك موظفان نفس الكود).

---

### آلية عمل الأكواد — التدفق الكامل

#### التفعيل

```
الموظف يرسل: "health_on_1234"
        ↓
n8n Webhook يستقبل الرسالة
        ↓
POST /api/v1/bot/check-auth
{
  "phoneNumber": "203066120912949@lid",   ← WhatsApp LID أو رقم الهاتف
  "activationCode": "health_on_1234"      ← محتوى الرسالة
}
        ↓
الخادم يبحث في جدول bot_users عمن لديه activationCode = "health_on_1234"
        ↓
إذا وُجد + الأمان تحقق → isBotActive = true، تسجيل اللid
        ↓
الرد: { "action": "activated", "full_name": "محمد أحمد الخطيب" }
        ↓
n8n يرسل رسالة ترحيب عبر واتساب
```

#### الرسائل العادية (البوت مفعَّل)

```
الموظف يرسل: "ما هو راتبي؟"
        ↓
POST /api/v1/bot/check-auth
{
  "phoneNumber": "203066120912949@lid",
  "activationCode": "ما هو راتبي؟"
}
        ↓
الخادم يجد المستخدم عبر اللid المسجَّل
        ↓
isBotActive = true + لم تمرّ 5 دقائق → يُحدِّث lastInteraction
        ↓
الرد: { "action": "message" }
        ↓
n8n يرسل الرسالة للذكاء الاصطناعي (Gemini)
        ↓
الذكاء الاصطناعي يُجيب → n8n يرسل الرد
```

#### الإيقاف اليدوي

```
الموظف يرسل: "health_off_1234"
        ↓
POST /api/v1/bot/check-auth
{
  "phoneNumber": "203066120912949@lid",
  "activationCode": "health_off_1234"
}
        ↓
الخادم يجد المستخدم: deactivationCode = "health_off_1234" → تطابق
        ↓
isBotActive = false
        ↓
الرد: { "action": "deactivated" }
        ↓
n8n يرسل رسالة وداع
```

#### انتهاء الجلسة تلقائياً (5 دقائق)

```
الموظف لم يرسل رسالة لمدة 5 دقائق
        ↓
الخادم يجد المستخدم (via LID)
        ↓
isBotActive = true + مرّ أكثر من 5 دقائق منذ lastInteraction
        ↓
isBotActive = false (تلقائي)
        ↓
الرد: { "action": "auto_deactivated" }
        ↓
n8n يرسل رسالة: "انتهت جلسة البوت بسبب الخمول"
```

**إذا أرسل الموظف رسالة بعد انتهاء الجلسة (وكان LID مسجَّلاً):**
```
الموظف يرسل أي رسالة
        ↓
الخادم يجد المستخدم (via LID)
        ↓
isBotActive = false (الجلسة منتهية)
        ↓
الرد: { "action": "auto_deactivated" }   ← يُعيد للمستخدم إشعار الانتهاء
        ↓
n8n يرسل: "انتهت الجلسة، أرسل كود التفعيل لإعادة التشغيل"
```

---

### خوارزمية `check-auth` — الخريطة الكاملة

```
استقبال { phoneNumber, activationCode }
          ↓
هل phoneNumber يطابق whatsappLid مسجَّل في bot_users؟
          ↓
    نعم ────────────────────────────────────────────────
          ↓
    هل activationCode = deactivationCode للمستخدم؟
          نعم → action: "deactivated" (إيقاف يدوي فوري)
          لا  ↓
    هل isBotActive = true؟
          ↓
        نعم ──────────────────────────────
              ↓
        هل مرّ أكثر من 5 دقائق؟
              نعم → action: "auto_deactivated"
              لا  → action: "message"
          ↓
        لا ──────────────────────────────
              ↓
        هل activationCode = activationCode للمستخدم؟
              نعم → action: "activated"
              لا  → action: "auto_deactivated"  ← [إشعار بانتهاء الجلسة]
          ↓
    لا ────────────────────────────────────────────────
          ↓
    هل هو LID غير معروف؟ → ابحث عبر الكود
          ↓
    هل activationCode يطابق activationCode أحد المستخدمين؟
          نعم → فحص أمان الهاتف/LID → action: "activated"
          لا  ↓
    هل activationCode يطابق deactivationCode أحد المستخدمين؟
          نعم → فحص أمان الهاتف/LID → action: "deactivated"
          لا  → action: "unauthorized" (صامت)
```

---

### حماية سرقة الجلسة (LID Hijacking)

عندما يحاول شخص B استخدام كود موظف A من جهاز مختلف:

```
شخص B يُرسل كود تفعيل الموظف A من جهازه
        ↓
check-auth: البحث بالكود → يجد الموظف A
        ↓
هل للموظف A whatsappLid مسجَّل؟
        نعم → هل يطابق LID الطلب الوارد؟
              لا → action: "unauthorized" (رفض صامت)
              نعم → مقبول (نفس الجهاز)
        لا  → قبول الطلب وتسجيل اللid الجديد
```

**إعادة تعيين اللid:** المدير يضغط أيقونة الهاتف بجانب الموظف في لوحة التحكم → يُمسح اللid → الموظف يستطيع التفعيل من جهاز جديد.

### أين تُستخدم الأكواد في البرنامج

#### في الخادم (`server/routes.ts`)

| الموضع | الكود المقابل | الغرض |
|--------|--------------|-------|
| `getBotUserByLid(incomingLid)` | — | إيجاد المستخدم عبر جهازه |
| `incomingCode === botUser.deactivationCode` | كود الإيقاف | التحقق من طلب الإيقاف |
| `incomingCode === botUser.activationCode` | كود التفعيل | التحقق من طلب التفعيل |
| `allBotUsers.find(u => u.activationCode === incomingCode)` | كود التفعيل | بحث عالمي للمستخدمين غير المعروفين |
| `allBotUsers.find(u => u.deactivationCode === incomingCode)` | كود الإيقاف | بحث عالمي عند الإيقاف |

#### في ورك فلو n8n

| العقدة | دورها مع الأكواد |
|--------|----------------|
| `Verify_Identity` | يرسل محتوى الرسالة كـ `activationCode` لـ `check-auth` |
| `Route_by_Action` | يقرأ الـ `action` الراجع ويوجّه |
| `WA_Welcome` | تُرسَل عند `action = "activated"` |
| `WA_Goodbye` | تُرسَل عند `action = "deactivated"` |
| `WA_AutoTimeout` | تُرسَل عند `action = "auto_deactivated"` |
| `AI_Agent` | يُفعَّل عند `action = "message"` |

#### في واجهة المستخدم

| الصفحة | الموضع |
|--------|--------|
| **المستخدمون → مستخدمو البوت** | إنشاء وتعديل الأكواد |
| **عمود "كود التفعيل"** | عرض الكود الحالي |
| **عمود "كود الإيقاف"** | عرض الكود الحالي |
| **أيقونة الهاتف** | إعادة تعيين اللid |

## ملخص سريع

| العنصر | القيمة / الموضع |
|--------|----------------|
| **مفتاح n8n (machine)** | `3477e2bd6616a95eb2dcbb3a9e39b663fddab5a90fe7d71cdd45a7b34040fca4` |
| **مفتاح admin (human)** | لا يوجد حالياً — أنشئ من الإعدادات |
| **أكواد التفعيل/الإلغاء** | تُحدَّد يدوياً لكل موظف في صفحة المستخدمين |
| **المصادقة عبر API** | `x-api-key` header أو `?_t=` query |
| **المصادقة عبر المتصفح** | جلسة + كوكيز |
| **مدة جلسة المتصفح** | **5 دقائق** متجددة |
| **مدة جلسة البوت** | **5 دقائق** خمول |
| **ملف التحقق** | `server/apiKeyAuth.ts` |
| **ملف منطق البوت** | `server/routes.ts` (سطر 1314+) |
| **جدول قاعدة البيانات** | `api_keys` (مفاتيح النظام)، `bot_users` (أكواد البوت) |
